Design Section 2: Component Design Language

How individual fields look and feel — the building blocks of every sheet

Core Principle: Inline Fields with Left-Border Accents

Every field follows the same visual DNA: a compact row with a subtle translucent background, a colored left border when relevant, inline label on the left, value on the right. No Vuetify v-card wrappers. No floating labels. Just data.

Design Rules

Field Height: 28-32px

Every simple field (string, number, boolean, choice) fits in ~30px height. No floating labels — inline label left, value right. This is roughly half the current Vuetify field height.

Translucent Backgrounds

Fields use rgba(255,255,255,0.04) backgrounds. The page background pattern shows through subtly. No opaque white/gray boxes. Fields feel like they belong to the sheet, not pasted on top.

Left-Border Color Accent

When a field has a color: parameter, it shows as a 3px left border. This gives visual grouping without heavy card wrappers. Resources get their color automatically from the resource type (HP = red, etc).

Section Headers: Minimal

Uppercase, small, slightly translucent, with a thin bottom border. No background. No card wrapping. They're orientation markers, not visual barriers.
